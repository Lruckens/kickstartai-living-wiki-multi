The Living Wiki Project Goal: Kickstart AI is attempting to solve a knowledge management problem where documentation (meeting notes, design decisions, code) from multiple concurrent AI projects is scattered (00:09:16). The solution, called the "living wiki," aims to be a system that continuously ingests project documents to produce a living, auto-updating knowledge base. This system should synthesize, summarize, detect changes, and surface connections, effectively functioning as a wiki that rewrites and expands itself as new information arrives (00:11:05).

Technical Components of the Living Wiki: The project involves building fi ve main components, which were detailed in a provided two-pager. These components include an ingestion pipeline to handle various document types (including PDFs), a generation engine to create structured wiki pages, a generator module for periodic outputs, a gap detector to identify missing or underdocumented information, and a permission layer. The team was also advised that an evaluation framework should run throughout the project (00:12:19). 

Project Implementation and Evaluation Strategy: The project's fi rst task is to document itself, which is intended to simplify the process by avoiding the need to share sensitive data and providing an intuitive evaluation method. The permission layer does not need to be a fully working architecture due to the short timeline, but the students must design how it would handle sensitive data from the beginning (00:15:20).
