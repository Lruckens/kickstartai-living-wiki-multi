# Deliverable: Evaluation Framework

**Last updated:** 2026-06-19
**Status:** scoped

## Summary
A framework measuring coverage, freshness, accuracy, and usefulness of the wiki, incorporating human-in-the-loop feedback to improve generation quality over time.

## Details
See the concept page [[evaluation-framework]] for the four dimensions. The blog post draft from [[generator-module]] is to be evaluated against a human-written version to make the creativity/opinion angle measurable. Requires a human baseline document that does not yet exist — see [[_gaps]].

KickstartAI's existing public technical and domain blogs (see [[kickstartai-blog]]) are a *candidate* corpus for the required human-written baseline — they are real human-authored articles in KickstartAI's voice. This is a possibility only; no specific article has been selected or ingested as the baseline. See [[_gaps]].

### Production approach (supervisor kickoff, 2026-04-16)
This is enumerated as **Sub-deliverable 6 (Evaluation Method)**. The team agreed that **each member selects a *different* evaluation method**, conducts **individual research** on it, and includes the results in **their own thesis** (see [[supervisor-kickoff-2026-04-16]]). This corroborates the wiki's reading of evaluation as a cross-cutting framework that "runs throughout" — produced via per-member individual research rather than a single shared build module. See [[user-journeys]], [[evaluation-framework]].
