## Details
### Problem statement (verbatim)
- Project knowledge lives in people's heads and scattered documents.
- Status updates require someone to stop and compile them manually.
- **Learnings from one project rarely reach the next one.** (A new, explicit articulation of the cross-project reuse motivation — see [[_reuse]].)

### The Living Wiki concept (verbatim framing)
- "A knowledge base that updates itself."
- Inspired by Andrej Karpathy's LLM Wiki concept.
- **"The first project this system documents is itself."**
- "Starts with KickstartAI docs and lab materials on day one."
- "Grows as you work – every PR, meeting note, and decision feeds in."
- "By the end: a complete auto-generated record of how it was built."

### What we're asking you to build (five modules, verbatim)
1. Ingestion pipeline (see [[ingestion-pipeline]])
2. Wiki generation engine (see [[wiki-generation-engine]])
3. Generator module (see [[generator-module]])
4. Gap detector (see [[gap-detector]])
5. Permission layer (see [[permission-layer]])

The **evaluation framework is *not* named in this build list** — consistent with the kickoff notes' framing that evaluation "runs throughout" the project rather than being a standalone build module (see [[evaluation-framework]]). The wiki therefore tracks **6 deliverables = 5 build modules + 1 cross-cutting framework** — see [[_gaps]].
